Syntax Guide

Write a play like a play.

Downstage is a plaintext format for stage plays. It keeps the source readable, adds just enough structure for tooling, and stays closer to what playwrights actually write than screenwriting formats do.

What Downstage is for

Readable source

The raw file should still look like a script, not a pile of tags.

Theatre-first syntax

Acts, scenes, dialogue, stage directions, songs, and character lists are native concepts.

Tool-friendly structure

LSP support, validation, parsing, and PDF rendering all hang off the same simple text format.

1

Document Structure

A Downstage document has an optional title page and then the play body. The body can contain a character list, acts, scenes, prose sections, dialogue, songs, comments, and page breaks. 

ALICE
Hello, world!

That minimal file is valid. No title page or headings required.

2

Title Page

Title-page metadata lives at the top of the file as Key: Value pairs. Any key name is accepted. Indented lines continue the previous value. 

Title: The Last Curtain Call
Subtitle: A Drama in Two Acts
Author: Eleanor Vance
Date: 2025
Notes: Inspired by true events
  and several missed cues.

The title page ends when the file hits a heading, a page break, or a non-indented line that is not a metadata pair.

3

Dramatis Personae

The character list is a dedicated section. Entries can include descriptions, aliases, and optional subgroup headings.

Accepted headings: # Dramatis Personae, # Cast of Characters, and # Characters all start the same section type. 
# Dramatis Personae

MARGARET — An aging actress
JAMES/JIM — Her estranged son
[JAMES/JIM]

## The Crew

STAGEHAND 1 — Quiet, efficient
STAGEHAND 2 — Nervous and chatty
  • NAME or NAME — Description are both valid.
  • NAME/ALIAS defines a shorter dialogue alias inline.
  • [FULLNAME/ALIAS] defines the same alias on a separate line.

4

Acts and Scenes

Acts are usually ## headings. Scenes are usually ### headings, though a non-act ## heading inside an act also becomes a scene. 

## ACT I
## ACT 2
## ACT: Finale
### SCENE 1
### SCENE 1: The Palace
### A Bare Stage

If you do not need acts or scenes, skip them. Downstage does not force a structure you are not using.

5

Dialogue

Dialogue starts with a character name on its own line, followed by an optional parenthetical and one or more speech lines. 

HAMLET
(aside)
To be, or not to be, that is the question.

MARGARET
They used to fill every seat, you know.
Every last one.

Character names are uppercase and may contain spaces, punctuation like periods and apostrophes, digits, and slashes for alias-aware names.

6

Stage Directions

Standalone stage directions start with >. Inline parentheticals inside dialogue stay part of the speech block. 

> The lights dim.
> CLAIRE crosses downstage.
HAMLET
To be (pause) or not to be.

7

Verse

Indent dialogue lines by two or more spaces to preserve line breaks as verse. 

MARGARET
  To stand upon this stage once more,
  to feel the boards beneath my feet,
  to hear the hush before the roar,
  to know this moment, bittersweet.

Prose and verse can be mixed inside the same speech block.

8

Songs

Songs wrap a block of theatrical content between SONG and SONG END. 

SONG 1: The Stagehand's Lament

JIM
  Under the lights we never stand,
  behind the curtain, close at hand.

SONG END

Numbering and titles are optional. The contents can include dialogue, verse, and stage directions.

9

Inline Formatting

Inline formatting works in dialogue, verse, and stage directions.

Syntax Result
*text* Italic
**text** Bold
***text*** Bold italic
_text_ Underline
~text~ Strikethrough

10

Comments and Page Breaks

// A line comment

/* A block comment
   spanning multiple lines */
===

### SCENE 2

Comments are ignored when rendering, so they will not appear in the PDF. === inserts a page break in rendered output.

11

Forced Elements

When the parser would otherwise classify a line incorrectly, force it. 

@horatio
My lord, I came to see your father's funeral.
.The Next Evening
  • @name forces a character cue.
  • .Heading forces a structural heading.

13

Tooling

Downstage ships with a parser, validator, PDF renderer, and LSP server. 

downstage parse play.ds
downstage validate play.ds
downstage render play.ds
downstage render --style condensed play.ds
downstage lsp

Source, releases, and editor setup live in the main repository.
